Introduction

Universal Sentence Encoder is a tool to extract feature vectors from long texts. Basically, it trains a model to produce feature vectors from any sentence in any language. The training data is incredibly huge, because it trained on publicly available webpages. It enables for example a language model trained on a lot of texts in one language to also generalize to text in other languages (like French, Chinese, and Russian for example), as long as those texts are in the same semantic domain.

What are the potential applications?

The benefits of using it is, of course, that it is not necessary to generate labels and train a model for each new domain. Also, the resulting vector is incredibly small, and that can make learning with it much more efficient. The system is also more flexible, for example if you want to test a new sentence, you don't need to create a new model for it, because it can get the vector from the encoder directly. It's nice to use such a system when you are creating an exploratory tool, as you don't need to know anything about the models' inner workings. You just give it a sentence in one domain, and it spits out the corresponding vector representation.

How this algorithm works? It works by training a deep recurrent neural network. You give it a sentence and it tries to predict the next word in the sentence (let's call it a token). The sequence of predicted tokens is then used to predict a text representation. This means that the network not only looks at individual words, but also looks at the context of these words. Thus it learns to extract the semantic meaning of a sentence. It's also trained in an unsupervised fashion, i.e., it doesn't need the corresponding labels, just like how word2vec is. As the dataset size is large enough, the embedding already contains useful information. The result is a model that can be trained on text in one language and be used for another language. The model's weights are language independent, so it's language agnostic in the sense that it doesn't know anything about how words are represented in a specific language.

Regression

(Image source: A Friendly Introduction to Machine Learning)

Regression for time series involves finding/designing a good feature set ($X$) that predicts our target values ($Y$). This process is known in machine learning as feature engineering.

# ts -> FeatureExtraction -> X (features)
X = FeatureExtraction.fit_transform(ts) # backwards information or not!

Lots of algorithms are widely used in the context of time series, including Linear Regression and tree-based ensembles such as Random Forest or Gradient Boosting.

from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import train_test_split
from sklearn.pipeline import make_pipeline
from sktime.forecasting.model_selection import temporal_train_test_split

# X, y

Transforming a regressor into forecaster:

from sktime.forecasting.base import ForecastingHorizon
from sktime.forecasting.compose import make_reduction
from sktime.forecasting.model_selection import temporal_train_test_split

forecaster = make_reduction(
    regressor, scitype="time-series-regressor", window_length=12
)
forecaster.fit(y_train)

fh = ForecastingHorizon(y_test.index)

Take aways

  • Interpolation, Regression and Forecasting are techniques that use diferent methods to make predictions;
  • Our world is chaotic thus your time series forecasting task may be more complex (multivariate, etc);
  • Model evaluation is crucial, including baseline analysis;
  • Validation may require the support of a domain expert that can interpret the results;
  • Applications can build expert systems (agents) that use predictions to act automaticaly (predictive controllers).
